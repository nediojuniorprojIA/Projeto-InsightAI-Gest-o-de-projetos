## 🎯 Visão Geral

Este repositório contém material teórico e prático sobre **Gestão de Projetos**, cobrindo desde conceitos fundamentais até metodologias modernas aplicáveis a diferentes contextos organizacionais.

### Objetivo Principal
Fornecer um guia completo para profissionais que desejam:
- Entender os princípios da gestão de projetos
- Aplicar metodologias reconhecidas (PMBoK, Agile, Lean)
- Utilizar ferramentas e frameworks práticos
- Melhorar a eficiência e entrega de projetos

## 📚 O que é Gestão de Projetos?

### Definição
Gestão de Projetos é a aplicação de conhecimentos, habilidades, ferramentas e técnicas às atividades do projeto para atender aos seus requisitos.

### Características Principais

| Aspecto | Descrição |
|---------|-----------|
| **Temporário** | Tem início, meio e fim definidos |
| **Único** | Produz resultado ou serviço único |
| **Objetivo Claro** | Busca atingir metas específicas |
| **Recursos Limitados** | Opera com orçamento e prazo definidos |
| **Stakeholders** | Envolve múltiplas partes interessadas |

## 🔄 Metodologias

### 1. Waterfall (Cascata)

**Características:**
- Linear e sequencial
- Planejamento detalhado no início
- Mudanças são custosas
- Ideal para projetos com requisitos bem definidos

**Fases:**
1. Requisitos
2. Design
3. Implementação
4. Teste
5. Deployment
6. Manutenção

**Vantagens:**
✅ Documentação clara e completa
✅ Fácil rastreabilidade
✅ Previsão de custos e prazos

**Desvantagens:**
❌ Inflexível a mudanças
❌ Feedback tardio
❌ Risco elevado no final do projeto

---

### 2. Agile

**Filosofia:** Entregar valor continuamente com iterações curtas

**Manifesto Ágil:**
- Indivíduos e interações > Processos e ferramentas
- Software funcionando > Documentação abrangente
- Colaboração com cliente > Negociação de contrato
- Responder a mudanças > Seguir um plano

**Metodologias Ágeis:**

#### **Scrum**

PRODUCT BACKLOG ↓ SPRINT PLANNING (4h) ↓ SPRINT (2 semanas) ├─ Daily Standup (15min) ├─ Development └─ Testing ↓ SPRINT REVIEW (2h) ↓ SPRINT RETROSPECTIVE (1.5h) ↓ INCREMENT PRONTO

**Papéis Scrum:**
- **Product Owner** - Define prioridades
- **Scrum Master** - Remove impedimentos
- **Time de Desenvolvimento** - Entrega o trabalho

#### **Kanban**

TO DO → IN PROGRESS → REVIEW → DONE

**Princípios:**
1. Visualizar o trabalho
2. Limitar WIP (Work In Progress)
3. Gerenciar fluxo
4. Implementar políticas de processo
5. Usar loops de feedback

#### **XP (Extreme Programming)**
- Pair programming
- Test-Driven Development (TDD)
- Integração contínua
- Refatoração
- Feedback rápido

---

### 3. Lean

**Princ��pios Lean:**
1. **Eliminar desperdício** - Remover atividades sem valor
2. **Amplificar aprendizado** - Rápido feedback
3. **Tomar decisões tarde** - Manter flexibilidade
4. **Entregar rápido** - Tempo-to-market
5. **Empoderar o time** - Autonomia
6. **Construir integridade** - Qualidade
7. **Otimizar todo** - Visão sistêmica

---

### 4. Hybrid

Combinação de metodologias para otimizar resultados:

HYBRID = Waterfall (Planning) + Agile (Execution) + Lean (Optimization)

## 🎨 Lean Canvas InsightAI

### O que é Lean Canvas?

Ferramenta simplificada para validar ideias de negócio em 1 página, desenvolvida por Ash Maurya como evolução do Business Model Canvas.

### Estrutura (9 Blocos)
┌─────────────────────────────────────────────────────┐ │ 1. PROBLEMA │ 2. SOLUÇÃO │ 3. PROPOSTA │ │ Principais problemas │ Ideias para │ Valor único │ │ do segmento │ resolver │ para │ │ │ │ clientes │ ├─────────────────────────────────────────────────────┤ │ 4. CHANNELS │ 5. REVENUE │ 6. MÉTRICA │ │ Canais de │ Modelos de │ Indicador │ │ comunicação │ receita │ de sucesso │ ├─────────────────────────────────────────────────────┤ │ 7. CUSTO │ 8. SEGMENTO │ 9. VANTAGEM │ │ Estrutura de │ Cliente │ Competitiva │ │ custos │ alvo │ Injusta │ └─────────────────────────────────────────────────────┘

### Preenchimento Estratégico

#### **1. Problema**
- Identifique 3 principais dores
- Exemplo: "Demora em análise de dados"

#### **2. Segmento de Clientes**
- Persona bem definida
- Exemplo: "Gerentes de Projeto em startups"

#### **3. Proposta Única de Valor**
- O que torna sua solução diferente
- Exemplo: "Análise em tempo real com IA"

#### **4. Solução**
- Como você resolve o problema
- Foco em MVP (Mínimo Produto Viável)

#### **5. Canais**
- Comunicação e distribuição
- Exemplo: LinkedIn, webinars, partnerships

#### **6. Fluxo de Receita**
- Modelo de negócio
- Exemplo: SaaS, freemium, licensing

#### **7. Métricas de Sucesso**
- KPIs mensuráveis
- Exemplo: CAC (Customer Acquisition Cost)

#### **8. Vantagem Competitiva Injusta**
- Difícil de copiar
- Exemplo: Dados exclusivos, timing, equipe

#### **9. Custos**
- Despesas operacionais
- Exemplo: Infraestrutura, salários, marketing

## 🔧 Processos Fundamentais

### 1. Iniciação do Projeto

**Atividades:**

└─ Definir Necessidade ├─ Identificar Stakeholders ├─ Criar Termo de Abertura └─ Validar Alinhamento Estratégico


**Saídas:**
- ✅ Termo de Abertura
- ✅ Registro de Stakeholders
- ✅ Baseline de Projeto

---

### 2. Planejamento

**Estrutura:**

Planejar Escopo └─ Definir limites, restrições
Coletar Requisitos └─ Detalhamento com stakeholders
Definir Atividades └─ Work Breakdown Structure (WBS)
Estimar Duração └─ Timeline realista
Sequenciar Atividades └─ Dependências
Estimar Recursos └─ Pessoas, materiais, equipamentos
Estimar Custos └─ Orçamento detalhado
Planejar Qualidade └─ Padrões e critérios
Planejar Comunicação └─ Frequência e meios
Planejar Riscos └─ Identificação e mitigação


---

### 3. Execução e Monitoramento

**Ciclo Contínuo:**

PLANEJAR → EXECUTAR → MONITORAR → CONTROLAR → AJUSTAR ↑ ↓ └───────────────────────────────────────────┘


**Atividades de Monitoramento:**
- 📊 Relatórios de Progresso
- 📈 Análise de Variância
- 🎯 Acompanhamento de KPIs
- 🚨 Gestão de Mudanças
- 📋 Atualizações de Registro de Riscos

---

### 4. Encerramento

**Checklist:**
- ✅ Aceitar Entregáveis
- ✅ Documentar Aprendizados
- ✅ Liberar Recursos
- ✅ Arquivar Documentação
- ✅ Realizar Retrospectiva
- ✅ Comunicar Conclusão
- ✅ Fechar Contratos

**Documento Final:**
- Relatório de Encerramento
- Lições Aprendidas (Lessons Learned)
- Recomendações para Projetos Futuros

## 📈 Indicadores de Desempenho (KPIs)

### KPIs de Projeto

| KPI | Fórmula | Alvo |
|-----|---------|------|
| **Schedule Performance Index (SPI)** | EV / PV | > 1.0 |
| **Cost Performance Index (CPI)** | EV / AC | > 1.0 |
| **Earned Value** | % Completo × Orçamento | Conforme cronograma |
| **Budget Variance** | EV - AC | ≥ 0 |
| **Schedule Variance** | EV - PV | ≥ 0 |
| **Variance at Completion** | BAC - EAC | Minimizar |

**Legenda:**
- **EV** = Earned Value (Valor Agregado)
- **PV** = Planned Value (Valor Planejado)
- **AC** = Actual Cost (Custo Real)
- **BAC** = Budget at Completion
- **EAC** = Estimate at Completion
